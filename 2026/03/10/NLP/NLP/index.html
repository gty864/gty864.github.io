Lecture1: Text Normalization

Word Tokenization

Space-based tokenization

即利用空格(和标点符号)分词

e.g.
I love you，分成 I, love, you 三个token

但这样遇到符号多的，有hyfen的，数字字母混杂的文本就不好处理了，所以要用正则表达式：

正则表达式

正则元字符:

  • \w ：匹配字母、数字、下划线
  • \d ：匹配数字
  • \s ：匹配空白符（空格、制表符、换行符等）
  • [A-Z] ：匹配大写英文字符（注意：正确写法是 A-Z 而非 A,Z）
  • [0-9] ：匹配数字（等价于 \d）

数量控制符:

  • + ：匹配前面的字符 1 次或多次（等价于 {1,}）

  • * ：匹配前面的字符 0 次或多次（等价于 {0,}）

  • ? ：匹配前面的字符 0 次或 1 次（等价于 {0,1}）

  • {n} ：匹配前面的字符正好 n 次

  • {n,m} ：匹配前面的字符 n 到 m 次（包含 n 和 m）

  • \. 代表 .这个字符

  • . 代表任意字符

  • \$ 代表 $这个字符

  • 单个 $ 代表终止字符

例子

Pattern（匹配模式）：\$?\d+(\.\d+)?%?
Output（匹配结果）：可匹配 12.5%、$8.2、$134、$152.3% 等格式（符号和小数部分按需存在）

模式拆解（帮助理解）：
  • \$? ：匹配 0 个或 1 个美元符号 $（? 表示 0/1 次）
  • \d+ ：匹配 1 个及以上数字（整数部分，+ 表示 1+ 次）
  • (\.\d+)? ：匹配 0 个或 1 个小数部分（\. 匹配小数点，\d+ 匹配小数位，整体用 () 分组后加 ? 表示可选）
  • %? ：匹配 0 个或 1 个百分号 %

Subword Tokenization (BPE)

对于中文这类无空格分隔的语言，正则表达式无法有效分词，因此需要其他解决方案：

  • 方案1：序列标注 + 监督学习
  • 方案2：Subword Tokenization（子词分词）

示例：将 postprocessing 切分为 post, process, ing 三个 token

字节对编码 (Byte-Pair Encoding, BPE)

核心流程

  1. 将语料库（corpus）中所有字符（含终止符）拆分，加入词汇表（vocabulary）；
  2. 统计出现频率最高的相邻字符对 a 和 b，合并为 ab 并加入词汇表；
  3. 将文本中所有的 a 和 b 合并，重复上述操作 k 次。

具体示例

初始语料库：CR7 siu siu siuuu
初始词汇表：_(终止符), C, R, 7, s, i, u
初始token拆分：C R 7 _ s i u _ s i u _ s i u u u _

  1. 第一次合并：

    • 频率最高的相邻字符对：s,i（出现3次）
    • 新词汇表：_, C, R, 7, s, i, u, si
    • 新token：C R 7 _ si u _ si u _ si u u u _
      注：同频率下优先合并位置靠前的字符对，BPE未明确规定此规则，为自定义逻辑

  2. 第二次合并：

    • 频率最高的相邻字符对：si, u（出现3次）
    • 新词汇表：_, C, R, 7, s, i, u, si, siu
    • 新token：C R 7 _ siu _ siu _ siu u u _

  3. 第三次合并：

    • 频率最高的相邻字符对：siu, _（出现2次）
    • 新词汇表：_, C, R, 7, s, i, u, si, siu, siu_
    • 新token：C R 7 _ siu_ siu_ siu u u _

最终结论：siu_ 合并为完整token；CR7 和 siuuu 因出现频率不足，仍拆分为多个token。

Word Normalization（词归一化）

目标：将单词/token转换为标准格式

Case folding（大小写折叠）

将所有字母转换为小写形式。

Lemmatization（词形还原）

将所有词表示为词元（基础形式），基于词法分析（morphological parse），将词拆分为语素（morpheme），如词干（Stems）、词缀（affixes）。
示例：He is reading stories → He be read story

Stemming（词干提取）

粗暴地截断词缀以得到词干，是词形还原的简单/粗糙替代方案。

对比示例

原始句子：
This was not the map we found in Billy Bones's chest.

Case folding 结果：
this was not the map we found in billy bones's chest.

Stemming 结果：
Thi wa not the map we found in Billi Bone s chest.
注：粗暴截断导致 this→thi、was→wa，误将末尾s当作复数词缀去除

Lemmatizing 结果：
This be not the map we find in Billy Bones's chest.

Lecture2: Text Representing

Word Vectors

将每个单词表示在向量空间中，核心优势：

  1. 易于计算处理
  2. 能捕捉语义相似度（向量空间中距离越近，语义越相似）
  3. 可从数据中自动学习

Sparse Vector Representations（稀疏向量表示）

Co-occurrence matrices（共现矩阵）

  • 维度：|V| * |V|（|V| 代表词汇表中单词总数）
  • 取值：第i行k列的值 = 第i行单词与第k列单词在指定上下文窗口中共现的次数

示例：
三段文本：

  1. CR7 like siu
  2. I like siu
  3. siu is good

共现矩阵：

CR7 like siu I is good
CR7 0 1 1 0 0 0
like 1 0 2 1 0 0
siu 1 2 0 1 1 1
I 0 1 1 0 0 0
is 0 0 1 0 0 1
good 0 0 1 0 1 0

结论：CR7 和 I 的向量均为 [0,1,1,0,0,0]，说明二者语义相近。

上下文窗口说明

  • 实际应用中不会使用全部文本，通常采用小上下文（段落）或 window of ±k words（目标词前后k个单词）；
  • 单词共现定义：单词a出现在单词b的前后k个词范围内；
  • 窗口大小选择：
    • 小窗口（1-3）：向量更偏向句法（synactic） 特征；
    • 大窗口（4-10）：向量更偏向语义（semantic） 特征。

PMI (Pointwise Mutual Information，点互信息)

公式：

PMI(X,Y)=log2P(X,Y)P(X)P(Y)PMI(X,Y) = log_2{\frac{P(X,Y)}{P(X)P(Y)}}PMI(X,Y)=log2​P(X)P(Y)P(X,Y)​
  • P(X,Y)P(X,Y)P(X,Y)：联合概率，X和Y在文本中共现的概率,等于是X,Y共同出现次数除以表里所有词出现次数；
  • P(X),P(Y)P(X),P(Y)P(X),P(Y)：边际概率，X/Y单独出现在文本中的概率，P(X)等于是X出现次数除以表里所有词出现次数。

作用：平衡高频词（如“的”、“是”）的影响，让矩阵更能反映有意义的词对关系。

  • PMI > 0：两词正相关；
  • PMI = 0：两词相互独立；
  • PMI < 0：两词负相关。

PPMI (Positive PMI，正点互信息)

背景：巨大语料库中负相关PMI值不准确，且用户对负相关不敏感；
公式：

PPMI=max(PMI,0)PPMI = max(PMI, 0)PPMI=max(PMI,0)

作用：将负PMI值置0，仅保留正相关信息。

Dense Vector Representing（稠密向量表示）

稠密向量在实际 NLP 任务中表现优于稀疏向量，核心原因：

  1. 去噪：稠密向量去除了代表不重要信息的低阶维度，保留核心特征；
  2. 泛化性：对未知数据（未见过的词汇/文本）拥有更好的泛化能力；
  3. 效率：维度远小于稀疏向量，更容易训练和部署使用。

Static Word Embedding

核心代表：word2vec、GloVe
特点：每个单词的向量是固定的，不随上下文变化。

这里我们只介绍word2vec.

Word2Vec 详细原理

Word2Vec 是由 Google 提出的一个词嵌入模型，用于通过上下文信息学习词向量。其核心思想是：“上下文相似的单词，其向量表示也相似”。Word2Vec 模型通过训练数据中的上下文来学习单词的密集表示（即词向量）。它有两个核心架构：

  1. Skip-gram（跳字模型）
  2. CBOW（连续词袋模型）

1. Skip-gram（跳字模型）

核心逻辑

Skip-gram 模型的核心思想是：给定一个中心词，预测该词周围的上下文词。模型通过学习如何根据一个中心词来预测其上下文，从而得到每个词的嵌入表示。

适用场景

  • 在小规模语料上效果更好。
  • 能更好地处理低频词。

示例

假设有一个句子：CR7 likes to celebrate with siu，并且设置窗口大小为 2，那么选择 siu 作为中心词时，模型的任务是：

  • 中心词：siu
  • 上下文词：celebrate, with, to, likes

即 Skip-gram 模型的目标是通过 siu 来预测上下文单词 celebrate, with, to, likes。

2. CBOW（连续词袋模型）

核心逻辑

CBOW 模型的核心思想与 Skip-gram 相反：给定上下文词，预测中心词。即通过上下文词的表示来预测中心词。

适用场景

  • 在大规模语料库上训练时，CBOW 会比 Skip-gram 更高效。
  • 高频词的词向量质量较好。

示例

假设句子为：CR7 likes to celebrate with siu，如果我们选择 siu 为中心词，窗口大小为 2，那么任务是：使用上下文词 celebrate, with, to, likes 来预测中心词 siu。

3. 优化方法

在训练 Word2Vec 模型时，采用SGD。常见的优化方法有负采样 (Negative Sampling).

负采样（Negative Sampling）

在负采样中，模型并不是计算所有词汇的概率，而是选择一个“正样本”（即目标上下文词）和若干个“负样本”进行训练。负样本是从词汇表中随机选择的，并且被假设为不相关的词。模型的目标是最大化正样本的概率，同时最小化负样本的概率。

目标函数：

J(θ)=−log⁡σ(uoTvc)−∑i=1klog⁡σ(−uiTvc) J(\theta) = - \log \sigma(u_{o}^T v_{c}) - \sum_{i=1}^k \log \sigma(-u_{i}^T v_{c}) J(θ)=−logσ(uoT​vc​)−i=1∑k​logσ(−uiT​vc​)

其中：

  • P(co−occurance∣c,o)=σ(uoTvc)P(co-occurance|c,o) = \sigma(u_{o}^T v_{c})P(co−occurance∣c,o)=σ(uoT​vc​), 代表中心词为c，边缘词为o在同一上下文共同出现的概率
  • σ(x) \sigma(x) σ(x) 是 Sigmoid 函数；
  • k 是负样本的数量。

负采样通过减少计算复杂度（仅计算 k+1 次 Sigmoid 运算），使得模型在大型语料库和词汇表下更加高效。

4. 输出的词向量

通过 Skip-gram 或 CBOW 模型训练得到的词向量通常是 50、100、200、300 维的实数向量。与传统的稀疏向量表示（如独热编码）相比，Word2Vec 的词向量维度远小，且能够更好地捕捉词汇之间的语义关系。

  • 这些词向量具有较好的语义相似性：语义相似的词在向量空间中往往靠得很近。
  • 例如，词向量可以通过向量运算实现词义操作，比如：
    • “King” - “Man” + “Woman” ≈ “Queen”。

  • 特点：单词的向量随上下文变化（同一个词在不同句子中向量不同）；
  • 代表模型：ELMo、BERT、GPT 等；

Document Representation（文档表示）

Sparse Vector

采用 Term-Document Matrix

  • 维度：词汇数 × 文档数；
  • 取值：矩阵中第i行第j列的值 = 第i个单词在第j个文档中出现的次数。

示例：莎士比亚戏剧的词-文档矩阵

单词 As You Like It Twelfth Night Julius Caesar Henry V
battle 1 1 8 15
soldier 2 2 12 36
fool 37 58 1 5
clown 6 117 0 0

文档向量示例

  • 文档 As You Like It 的向量表示：[1, 2, 37, 6]（列向量）；
  • 注意：不建议用行向量表示单词（如 battle 的行向量 [1,1,8,15]），原因：
    1. 若文档数量少，词向量易过拟合；
    2. 高频词（如 the/it）在所有文档中计数都高，向量值相近，无法区分语义。

原始频率的问题

直接使用单词的原始出现次数（raw frequency）存在缺陷：

  • 高频无信息词（如 it、the、good）的计数会主导整个向量；
  • 这类词对文档语义区分无帮助，却因数量大掩盖了关键词的特征。

于是引入TF-IDF:
TF-IDF 是对原始词频的优化，核心思想：惩罚在所有文档中都高频出现的词，突出仅在少数文档中高频出现的词。

核心公式:

  1. 词频（Term Frequency, TF）
    表示单词 t 在文档 d 中的出现频率：

    tft,d=log10(count(t,d)+1)tf_{t,d} = log_{10}(count(t,d) + 1)tft,d​=log10​(count(t,d)+1)

    注：也有其他变体，如原始计数、归一化计数等

  2. 文档频率（Document Frequency, DF）

    dft=包含单词t的文档数量df_t = 包含单词t的文档数量dft​=包含单词t的文档数量

  3. 逆文档频率（Inverse Document Frequency, IDF）

    idft=log10(Ndft)idf_t = log_{10}(\frac{N}{df_t})idft​=log10​(dft​N​)

    其中 N = 语料库中的总文档数。

  4. 最终 TF-IDF 值

    w=tft,d×idftw = tf_{t,d} × idf_tw=tft,d​×idft​

稠密向量表示（Dense Vector）

文档的稠密向量表示主要有两种实现方式：

基于 SVD 的降维

Latent Semantic Analysis (LSA): 就是把SVD用在term-document matrix上，对 TF-IDF 矩阵进行SVD，保留主要奇异值对应的维度，将高维稀疏的 TF-IDF 矩阵转换为低维稠密矩阵；

  • 注意：这个LSA也可以被用于term-term matrix上

假设原矩阵大小是w*c，SVD后变成了U Σ\SigmaΣ V,大小分别是 w*k,k*k,k*c, 然后我们保留U的前k行和V的前k列，记为U1U_1U1​和V1V_1V1​，再和 Σ\SigmaΣ相乘，就得到了只保留保留主要奇异值的低维矩阵，大小是k*k的

基于神经网络的方法

适用于文档、短语、句子的稠密向量表示，核心思路是基于词嵌入构建文档嵌入，主要方法：

  1. 池化（Pooling）：对文档中所有单词的嵌入向量做均值池化（mean）、最大池化（max），直接得到文档向量；
  2. 边界词处理：在文档首尾添加特殊标记（如 <CLS>/<SEP>），用标记的向量表示整个文档；
  3. 序列模型：对词嵌入序列应用 LSTM、Transformer 等模型，通过编码得到文档的稠密向量（后续章节详细讨论）。

Lecture 3: Text classification

Text classification, 顾名思义就是对输入的文本进行分类，最简单的例子就是，输入是一段评论，需要输出这段评论是正面的(positive)还是负面的(negative)。

e.g

Ronaldo is the best football player -> + (positive)
Messi is better than Ronaldo, he is just a shooter. -> - (negative)

Definition:
Input:
A document 𝑑
A fixed set of classes 𝐶 = 𝑐1, 𝑐2, … , 𝑐𝐾
Output:
A predicted class c ∈ C

接下来，我们将介绍文本分类的方法，主要分为两大类，一种是传统的基于规则的方法(Rule-based methods). 另一种是机器学习的方法(machine learning methods)

Rule-based methods

这种方法需要专家来写具体的规则，比如文本中出现了“不错”这个词就代表是正面，出现了“不太好”这个词就代表是负面。

利用正则表达式去提取文本中的特殊单词。在Lecture1中我们提到了字符层面的正则表达式，而这里则是单词层面的正则表达式。

e.g

正面反馈的正则表达式：
.*((very)? good|great|awesome|not bad).*

负面反馈的正则表达式：
.*(bad|awful|worst|ridiculously overpriced).*

.代表任意字符，*代表0个或多个 ?代表0个或1个, a|b代表a或者b

问题来了，如果同时匹配正面反馈的单词和负面反馈的单词应该怎么办？

1.为每个词设置优先级，看谁的优先级高
2.直接使用逻辑表达式，例如，若同时满足出现 (a∨b)∧¬c(a \lor b) \land \neg c(a∨b)∧¬c,则将其分类到 c1c_1c1​. 表达的意思是：文本中若出现a或b且没有出现c,则分类到c1c_1c1​

Machine learning methods

机器学习方法一般分为两大类，一个是生成式分类器(generative classifiers)，一个是判别式分类器(discriminitive classifiers)

假设输入文本是d，类别是c

生成式学 P(d|c)P(c), 对数据直接建模
判别式直接学 P(c|d)， 以找差异为主

generative classifiers

我们为什么要算precision 和 recall，不直接用accuracy来评估呢:
知乎上有一个经典笑话，如何分辨孩子有没有数学天赋，很简单，直接将其分类为没有数学天赋，准确率高达99.9%. 用文本分类的思想来看，我们的分类器是直接分类为没有天赋，这是一个很差的分类器因为他不可能分辨出有天赋的孩子，但是如果光看准确率，我们却会认为这个分类器是一个很好的分类器，因为他准确率很高。如果这个时候我们去算recall的话，会发现这个分类器的recall是0，那就代表了这个分类器根本无法分类，只是单一归类。

比如，我们可以用K-means来对这些文本向量做聚类

简单说一下K-means的步骤，首先，我们先人为确定k的数量，即分成几个簇，然后，选取k个质心，可以随机选点，也可以用K-means++来选. 然后计算每个点到每个质心的欧式距离，把每个点分给离他最近的质心，最后形成k个簇，然后重新计算每个簇所有点的均值，将其取为新的质心。重复上面的操作直至收敛(即质心不再变化).

👉 对每个簇：

  • 找出这个簇中最多的真实类别

  • 统计“对的数量”

  • 越大越好（最大为 1）
  • 偏向于：
    • 簇越多 → purity 越高
    • 极端情况：
      • 每个点一个簇 → purity = 1

👉 对每个真实类别：

  • 找出它主要被分到哪个簇
  • 统计“被正确集中”的部分

  • 越大越好
  • 偏向于：
    • 簇越少 → inverse purity 越高

      • 所有点一个簇 → inverse purity = 1

Probabilistic language modeling(LM)主要是在干一件事，就是计算一句话(一组词)的概率。即 P(w1,w2,...wn)P(w_1,w_2,...w_n)P(w1​,w2​,...wn​) 的概率

显然也不可能，因为同样地，我们找不到能覆盖所有句子的语料库

接下来介绍三种能够计算这一概率的语言模型，分别是N-gram,RNN/LSTM,Transformer

简单概括一下这三种语言模型，刚刚所说的trivial的方法是直接去看前面所有的单词，然后预测下一个词的概率，这是不可能的。因此上一行提到的三个模型都做了一些改进，N-gram是只看当前预测单词的前N个单词，再之前的词直接舍掉不看了，RNN是看之前所有的单词，但是把他们的信息全都压缩了，transformer是基于注意力机制，看之前所有的单词，但是对一部分重要的单词更加注意，另一部分次要的单词就不那么注意。

然后再提一下如何处理unkown word，如果在测试中遇到了没在词表里出现的词应该怎么办？

首先，我们创建一个叫UNK的token， 代表未知单词。 我们可以在训练的时候就算出UNK token的概率。我们创建一个固定的词表，在text normalization部分的时候，把所有不在词表中的单词都标为UNK, 然后我们就可以把UNK当成一个正常的词来计算他的概率了。

当然也有一些其他的处理方法，比如在字母/词缀层面构建模型，去预测字母/词缀的概率。

RNN/LSTM

introduction

RNN是基于神经网络的一种语言模型，他相比于N-gram的优势在于：

  1. 可以看到前面所有的单词
  2. 相比于N-gram，计算量小

下面给出一张RNN的示意图

可以看到，h是hidden state，他是由前一个hidden state和当前词的word embedding经过线性变换后过了一个sigmoid函数得到的

training

RNN对于每一个hidden state h，都有一个输出y，代表用之前历史文本里的词预测下一个词的概率，那么对于单个y，我们的损失函数就可以是下一个词的正确值和预测值的交叉熵(cross entropy)

J(t)(θ)=CE(y(t),y^(t))=−∑w∈Vyw(t)log(y^w(t)) J^{(t)}(\theta) = CE(y^{(t)},\hat{y}^{(t)}) = -\sum_{w \in V}{y^{(t)}_w log(\hat{y}_w^{(t)})} J(t)(θ)=CE(y(t),y^​(t))=−w∈V∑​yw(t)​log(y^​w(t)​)

其中，y(t)y^{(t)}y(t)是真实值，是一个one-hot vector，比如说，下一个单词是jack，词表是[bill，jack，messi], 那么y(t)y^{(t)}y(t)就是[0,1,0], 只有jack的值是1，词表中其他的词全是0. 而y^(t) \hat{y}^{(t)}y^​(t) 是RNN输出的预测值，是整个词表里的词是下一个词的概率，他们做交叉熵，除正确词外所有值都是0，所以我们只要看正确值的-log就行了,即

J(t)(θ)=−logy^xt+1(t) J^{(t)}(\theta) = -log{\hat{y}_{x_{t+1}}^{(t)}} J(t)(θ)=−logy^​xt+1​(t)​

把所有预测值求平均，得出最终的损失函数:

J(θ)=1T∑t=1T−logy^xt+1(t) J(\theta) = \frac{1}{T} \sum_{t=1}^{T} -log{\hat{y}_{x_{t+1}}^{(t)}} J(θ)=T1​t=1∑T​−logy^​xt+1​(t)​

problem

RNN有一个很大的问题，那就是在训练的时候容易出现梯度消失或者梯度爆炸的问题，这是因为RNN的序列通常很长，而求前面的hidden state的梯度需要根据链式法则一层一层往回乘，如果乘的数都很小就会梯度消失，乘的数都很大就会梯度爆炸。

由于梯度消失的问题，RNN通常善于学习Sequential recency 而非 Syntactic recency。sequential recency 在预测时更看重离自己近的单词，而RNN由于梯度消失的缘故，在更新权重时更偏向于近处单词(梯度大)，而忽略远处单词(梯度小/消失)

Syntactic recency: The writer of the books is
更关注语法，发现主语是writer所以是is

Sequential recency: The writer of the books are
更关注顺序，books离预测词更近，所以是are

Due to vanishing gradient, RNN LMs are better at learning from sequential recency than syntactic recency, so they make this type of error more often than we’d like [Linzen et al 2016]

梯度爆炸有一个解决办法，那就是gradient clipping, 当我们发现梯度超过某个限度时，直接将其缩小(scale down)到安全值

但梯度消失不能这么做，因为你无法确定梯度趋近于0的时候是梯度消失还是已经收敛了

LSTM

LSTM(Long Short Term Memory) 是RNN的一个变种

如下图所示

LSTM对于每个hidden state 加了一个 cell state， 并且我们同时还维护三个gate，分别是forget， input，和output。 forget控制忘掉多少上一个cell的信息，input控制输入多少新cell (c~(t)\tilde{c}^{(t)}c~(t)) 的信息到当前cell中，output代表输出多少cell中的信息到hidden state中

LSTM能缓解梯度消失/爆炸的问题，这是因为传统RNN一直在对之前的hidden state进行线性或非线性的变换，这会让之前的信息越来越难保存下来，而LSTM通过增加遗忘门，输入门等门控，让我们能够人为的去控制应该保留哪些信息，遗忘哪些信息。

当然，LSTM不能保证消除了梯度消失/爆炸的问题，他只是提供了一个更简单学习长程依赖的方式。

Lecture 6: Sequence to Sequence

seq2seq, 就是输入一个sequence，输出一个sequence的任务。具体应用有：机器翻译，回答问题，概括文本等。

Methods

seq2seq的方法，与LM类似，主要有RNN，attention，transformer三种。

在这类任务中，我们有encoder用于分析输入sequence，还有decoder用于输出sequence。

如下图所示，这是用RNN处理seq2seq的示意图

先把英文作为输入，encoder结束后输入起始符，通过自回归decoder输出中文翻译内容

下图展示了利用attention处理seq2seq的示意图，可以看出，处理方法就是在当前token位置通过计算attention score，然后过一个softmax，得到词的概率分布，选出适合的，然后不断循环直至结束(即输出终止符)。

下图是transformer的encoder-decoder结构示意图，采用cross attention,即K，V矩阵采用encoder里的，而Q矩阵采用decoder里的

注意，这里encoder的multi-head attention部分不需要mask，因为这里不是LM，我们允许encoder看全部的内容.

Learning

训练数据: parallel corpus: paired input-output sequences
Objective: maximize the conditional likelihood P(y|x) (x is the input,y is the output)

e.g. Loss function = 对于输出sequence中每个token的预测正确概率取-log，再取一个平均

Teacher forcing: 模型每次预测下一个词时，都会给它提供正确的前一个词作为输入，而不是使用模型自己预测出来的词

但Teacher forcing的问题在于，模型训练时总是看到正确的前一个词，但在实际推理（生成）过程中，模型只能看到它自己的预测结果，而不是正确的词。这样，模型在训练时并没有看到自己的错误，导致了暴露偏差（Exposure Bias）

因此要用scheduled sampling(定时采样)。 与其在整个训练过程中始终使用正确的词作为输入，模型在训练中会有一定的概率使用它自己预测的词作为输入。这个概率从一开始较小（这样模型仍然能依赖正确的词），但随着训练的进行，使用预测词的概率会逐渐增大。这样，模型在训练过程中就能逐步体验到自己的预测（以及错误），有助于它更强大。随着时间的推移，模型越来越多地暴露于自己的错误中，从而提高了它在生成时的连贯性和鲁棒性。

Decoding

接下来讲一下decoding部分采样的一些方法：

Greedy decoding

在预测时，每步只取概率最大的词(argmax)

问题显而易见：无法撤销操作，一旦选错一个词，就会导致最终输出结果完全不一样

Beam search

在每一步都保留k个最好的部分sequence，k被称为beam size
如何判断最好？看分数：

score(y1,...,yt)=∑i=1tlogPLM(yi∣y1,...,yi−1,x) score(y_1,...,y_t) = \sum_{i=1}^{t} logP_{LM}(y_i|y_1,...,y_{i-1},x) score(y1​,...,yt​)=i=1∑t​logPLM​(yi​∣y1​,...,yi−1​,x)

示意图：

可以看到，k=2,因此我们只保留最好的两条路径，即score最高的，这里的话是a 和me的两条路径因为分最高被我们选中，于是我们的下一步就是以a和me为起点再向后找两个最好的，日后在那四条路径里面再去keep两条最好的，重复操作直至达到停止条件.

Beam search中，如果一条路径输出了end token，那就直接把他放在一边不管，对剩下的路径继续进行操作直至达到停止条件。停止条件是：我们reach了timestep T,或者我们已经有了n条输出了end token的路径.

那么在beam search结束之后，我们会有一堆完成的hypotheses，即一堆输出了end token的完成了的序列，我们应该选则哪一个？不应该选择分数最高的那个，因为如果纯看分数的话，序列短的占优，所以应该把score进行length normalization之后再根据score选分数最高的sequence

finalscore(y1,...,yt)=1t∑i=1tlogPLM(yi∣y1,...,yi−1,x) final_score(y_1,...,y_t) = \frac{1}{t} \sum_{i=1}^{t} logP_{LM}(y_i|y_1,...,y_{i-1},x) finals​core(y1​,...,yt​)=t1​i=1∑t​logPLM​(yi​∣y1​,...,yi−1​,x)

Beam search不保证一定能找到最优解,甚至不能保证一定比greedy decoding要好，但他大部分情况下都比greedy好

需要注意的是，beam search的 computational cost是几个方法里面最大的，因为他要保留k个最好的轨迹，但同时beam search的diversity也是没有后面几个sampling大的

Sampling

Sampling: 直接根据输出的概率分布来抽取token，大概率抽到概率最高的token，小概率抽到概率低的token

Top-k Sampling: 只找概率最大的k个token，在他们之中采样

Top-p Sampling(nucleus sampling): 找概率和大于等于p的最小token set，在这个set里面采样，比如说我有[0.1,0.3,0.5,0.1] 4个token的概率，p=0.7，那么直接选0.5和0.3的两个token，在他们之中采样

当然,top-k,top-p在选完之后还要normalize一下，保证选出来的这几个概率和为1

Temperature sampling: 通过调节temperature参数，小于1让分布更sharp，大于1让分布更smooth

Other desiderata：
Reduce repetition
1.Penalize generation of already-seen tokens
Can also be incorporated into training (unlikelihood objective)
2.Prevent attending to the same words

Avoid hallucination (generating things not in the input) for translation, summarization, etc. :
Multiple causes and solutions

Non-Autoregressive Decoding (NAT)

Encoder保持不变，在Decoder部分先预测生成的目标序列的长度k，通常是通过一个线性分类器来完成。然后输入k个特殊符号到解码器，并行地解码所有词，不需要mask of future tokens

优点: fast
缺点: low accuracy

一些改进: directed acyclic transformer, iterative NAT…

Lecture 7: Pretrained Language Models(PLMs)

在NLP里，之前的范式(paradigm)是training，对于每个不同的task，收集相关数据，训练一个对应的模型。

但这样做太繁杂了，于是我们引入了一个新的范式：pretraining language models，训练一个general-purpose model.而数据则直接用无标注文本(much more abundant)，learning objective采用LM

PLMs的三种形态: Encoder, Decoder, Encoder-Decoder

Encoder

encoder 可用于计算text representations,我们之前提到过static word representation,方法有LSA，Word2vec等，这种每个词只有一个固定的vector，而encoder可以提供基于上下文的embedding，即Contextualized Word Embeddings，
可通过上下文实时更新embedding.

e.g. 在RNN/transformer LM中，各个位置的hidden vector就是context-specific word representations at each position

ELMo：Embeddings from Language Models

Basic structure

训练一个正向 LSTM 语言模型和一个反向 LSTM 语言模型
1.使用 2 层 LSTM
2.使用字符 CNN 构建初始的词表示
3.使用残差连接
4.在两个 LSTM 中共享令牌表示和 softmax 词预测层的参数

The two layers of Elmo

ELMo 学会了结合不同层次的表示。这两个层次有不同的用途：

1.低层次 更适合处理低级语法等任务：如词性标注、句法依赖关系
2.高层次 更适合处理高级语义任务：如情感分析、语义角色标注、问答系统

How to use/train

使用 ELMo 执行任务：

运行 ELMo 对输入句子进行词表示生成。然后让（任何）最终任务模型使用这些表示。

训练最终任务模型时：

冻结 ELMo 的权重,或者同时训练 ELMo 的权重与最终任务模型的权重，通常这会带来更好的性能。通常使用较低的学习率来训练 ELMo 的权重。

This is the pretrain+finetune paradigm!
Pretrain: train the word embedding model on a general task with lots of data (e.g., LM)
Finetune: for a specific task, connect the word embedding model to the task model and continue its training together with the task model

BERT: Bidirectional Encoder Representations from Transformers

From ELMo to BERT
1.Replace LSTM with Transformer (Long-range dependency & parallelizability)
2.Use subword tokenization (WordPiece)(Similar to BPE)

BERT采用masked language model (MLM)，在句子中Mask k% 的词, 然后去预测 masked words。这是因为我们是要最终输出embedding，需要整个上下文的信息，而传统LM是单向的，是只能看到前文，后文看不到(防止模型作弊)。

MLM的细节: predict 15% of subword tokens, replace 80% of them with [mask], 10% random token, 10% unchange.

Why the last part?

No masking during fine-tuning and prediction
The model must learn to build strong representations of nonmasked words

}

通过这样的训练，模型会理解我们给他的任务，并执行这些任务

预训练模型可能会继续生成文本，因为他不知道“总结”是什么意思

模型会理解“总结”这个词的意思，并遵循格式给出结构化回答
